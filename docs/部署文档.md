# Grape Server 部署文档

## 1. 服务概述

Grape Server 是一个基于 Spring Boot 的后端服务，提供了 AI 相关的业务逻辑，包括测试用例生成、AI 模型调用等功能。

## 2. 环境要求

### 2.1 系统要求

- **操作系统**：Linux/Windows/macOS
- **Java 版本**：JDK 17
- **Maven 版本**：3.6.0+

### 2.2 硬件要求

- **CPU**：至少 4 核
- **内存**：至少 8GB
- **磁盘空间**：至少 50GB

## 4. 部署步骤

### 4.1 克隆代码

```bash
git clone <repository-url>
cd grape-server
```

### 4.2 配置外部服务

1. **MySQL 数据库**：
   - 创建名为 `grape` 的数据库
   - 确保用户 `root` 具有足够的权限

2. **Minio 对象存储**：
   - 创建名为 `grape` 的存储桶

3. **Qdrant 向量数据库**：
   - 确保服务正常运行

4. **科大讯飞星火大模型**：
   - 确保 API 密钥有效

5. **Ollama 本地 AI 模型**：
   - 确保服务正常运行
   - 确保 `deepseek-r1:1.5b` 模型已下载

### 4.3 配置应用

编辑 `src/main/resources/application.yml` 文件，根据实际环境修改配置：

- MySQL 数据库连接信息
- Minio 配置信息
- Qdrant 配置信息
- 科大讯飞星火大模型配置信息
- Ollama 配置信息

### 4.4 构建项目

```bash
mvn clean package -DskipTests
```

### 4.5 启动服务

```bash
java -jar target/grape-0.0.1-SNAPSHOT.jar
```

或者使用 Spring Boot 插件启动：

```bash
mvn spring-boot:run
```

## 7. 常见问题排查

### 7.1 数据库连接失败

- 检查 MySQL 服务是否正常运行
- 检查数据库连接配置是否正确
- 检查网络是否连通

### 7.2 Minio 连接失败

- 检查 Minio 服务是否正常运行
- 检查 Minio 配置是否正确
- 检查网络是否连通

### 7.3 Qdrant 连接失败

- 检查 Qdrant 服务是否正常运行
- 检查 Qdrant 配置是否正确
- 检查网络是否连通

### 7.4 AI 模型调用失败

- 检查科大讯飞星火大模型 API 密钥是否有效
- 检查 Ollama 服务是否正常运行
- 检查网络是否连通

### 7.5 服务启动失败

- 检查端口是否被占用
- 检查外部服务是否都已启动
- 检查配置文件是否正确

## 8. 监控和日志

### 8.1 日志配置

服务使用 Spring Boot 默认的日志配置，日志级别为：

- Spring Security：DEBUG
- com.grape.grape.mapper：DEBUG
- com.grape.grape.service.impl：DEBUG
- com.mybatisflex.core.mapper.Mapper：DEBUG
- com.mybatisflex.core.query.QueryWrapper：DEBUG

### 8.2 监控

服务集成了 Spring Boot Actuator，可以通过 `/api/actuator` 端点访问监控信息。

## 9. 安全注意事项

- 生产环境中应修改默认的数据库密码和 Minio 密钥
- 生产环境中应禁用 DEBUG 级别日志
- 生产环境中应配置防火墙，限制外部访问
- 生产环境中应使用 HTTPS 协议
